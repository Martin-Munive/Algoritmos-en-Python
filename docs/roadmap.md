# Hoja de ruta editorial

## Objetivo inmediato

Poner el proyecto en formato de libro web visible desde el inicio.

## Fase 1

1. Infraestructura `MkDocs + Material + GitHub Pages`
2. Auditoría editorial
3. Definición de arquitectura del libro

## Fase 2

1. Recuperar esencia, intención, índice y estructura del libro original.
2. Contrastar esa esencia con la auditoría editorial y con el contenido ya escrito.
3. Rediseñar la arquitectura publicable en MkDocs.
4. Migrar solo después de normalizar cada capítulo.
5. Reescribir o depurar capítulos para que funcionen como libro web, no como repositorio de notas.
6. Validar render, navegación y bloques de código en GitHub Pages.

## Fase 3

1. Expansión curricular
2. Casos aplicados
3. Publicación incremental y revisión continua

## Criterio de reinicio

El reinicio no significa descartar el contenido anterior. Significa usarlo como material fuente para construir una edición limpia, coherente y publicable.

Cada unidad nueva debe responder:
- que concepto enseña;
- por que aparece en ese punto del índice;
- que ejemplo médico o de ciencias de la vida lo justifica;
- que código debe ejecutar o ilustrar;
- que resultado debe ver el lector.
